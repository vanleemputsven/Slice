# Slice — architecture notes

## Inherited from IP-WaterView (conceptually)

- **`src/` layout** with App Router, feature-oriented `components/` folders, and `lib/` for domain logic and validation.
- **Semantic design tokens** mapped into Tailwind (canvas, surface, accent, card shadow)—WaterView uses light teal/aqua; Slice uses a dark neutral + coral accent while keeping the same token *idea*.
- **Zod at boundaries** for subscription records and preferences; no silent parse of localStorage without validation.
- **Product shell**: sticky header, `max-w-6xl` content column, composable cards (`.slice-card` / `.wv-card` pattern).

## Modernized choices

- **Next.js 16** + **Tailwind CSS v4** (`@import "tailwindcss"`, `@theme inline`) as generated by `create-next-app`, instead of Tailwind v3 + separate `tailwind.config.ts`.
- **Client-first persistence** with `localStorage` for MVP/demo portability—no Supabase/Prisma—while keeping validation and derive-only fields (e.g. “my share”) explicit in `lib/subscriptions/calculations.ts`.
- **React Hook Form + `@hookform/resolvers@3` + Zod 3** for add/edit flows (v5 resolvers target Zod 4; this pairing stays stable on Next 16). Native `<dialog>` for a lightweight modal.
- **Recharts** (`recharts`) for dashboard-only client visualizations; aggregates live in `lib/subscriptions/chart-data.ts` so charts stay presentation-only.

## Data model

- Subscriptions are **normalized records** (`SubscriptionRecord`); monthly burn uses **cycle normalization** (monthly / yearly / N-month custom) before share-splitting.
- **Share-Check**: when `shared` is true, `shareCount` is people in the split (including the user); personal monthly share is `(monthly total) / shareCount`.

## First-run behavior

- If no `localStorage` payload exists for subscriptions, the app **seeds demo data** so the UI is immediately realistic. Clearing data writes an **empty array** (not deleting the key) so demo data does not auto-return after a deliberate clear.
